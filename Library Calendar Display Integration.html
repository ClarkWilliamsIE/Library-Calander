      • Commit each file directly to main

    3. For the folder structure (like src/components) just include the slashes in the file name field (e.g. src/components/ScheduleTimeline.jsx)

  3. Store your API key and calendar IDs as secrets

    1. Go to Settings → Secrets and variables → Actions → “New repository secret”

    2. Add

      • Name: VITE_GCAL_API_KEY

      • Value: your Google Calendar API key

    3. Add another

      • Name: VITE_CALENDAR_IDS

      • Value: your comma-separated calendar IDs

  4. Add a GitHub Actions workflow for build and deploy

    1. Click “Add file” ▼ → “Create new file”

    2. Name it .github/workflows/deploy.yml

    3. Paste the following:

      yaml
      name: Build and Deploy
